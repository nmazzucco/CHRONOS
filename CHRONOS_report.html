CHRONOS Report

Stepwise illustrated report for modelling Neolithic obsidian production and circulation in the Central Mediterranean.

The figures are organised according to the analytical sequence of CHRONOS.R.

Executive summary

This report follows the logic of the analytical workflow: data cleaning, production-score ranking, Brainerd–Robinson diagnostics, rule-distance network modelling, source-filtered networks, and final site-location mapping.

1. Obsidian technological data

The workflow starts by reading and cleaning the full obsidian table and the two chronological subsets. Site names, coordinates, technological counts, total artefact counts, and source-attribution fields are standardised before analysis.

No figure is generated at this stage. The outputs are cleaned tables used by later ranking and network steps.

2. Calculate production-contribution values and production-score ranks

Each technological class is normalised by the total amount of that class across the dataset. The production score is calculated as the sum of the contribution values for blades, flakes, cores, and by-products/waste.

Production_score = blade.PN + flake.PN + core.PN + by.products.waste.PN

The rank thresholds are then derived from the production-score distribution: Rank 3 corresponds to the upper 10%, Rank 2 to the 80th–90th percentile range, and Rank 1 to all remaining sites. Raw-material source nodes are added manually as Rank 4.

3. Evaluate rank structure in Brainerd–Robinson dissimilarity space

The production-score ranks are evaluated in multivariate technological space using a Brainerd–Robinson-style dissimilarity matrix and PCoA visualisation.

4. Build rule-distance network models for two chronological intervals

For each chronological interval, the workflow evaluates three rule sets across distance thresholds from 100 km to 650 km. The report presents the two intervals separately, and within each interval it follows Rule A, Rule B, and Rule C.
