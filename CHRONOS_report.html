CHRONOS Report

Modelling Neolithic obsidian production and circulation in the Central Mediterranean

Illustrated technical report for GitHub publication, reproducibility documentation, and methodological transparency.

Executive summary

The CHRONOS workflow classifies obsidian assemblages by technological production involvement and models potential circulation pathways across the Central Mediterranean during two Neolithic chronological intervals: 6100–5500 cal BC and 5500–4900 cal BC.

Input data
Obsidian assemblage tables, coordinates, source-attribution columns, and a base map shapefile.
Ranking
Sites are classified using a reproducible production-score method based on technological contribution values.
Networks
Rule-distance models test different assumptions about obsidian circulation and connectivity thresholds.
Outputs
90 PNG figures plus CSV tables documenting rankings, metrics, centrality, and source-confirmed networks.

Analytical workflow

  1. Data loading and cleaning. The script reads the full dataset and the two chronological subsets from RAWDATA/, standardises column names, converts coordinates and counts to numeric values, and removes incomplete records.
  2. Production-score ranking. Each site contribution to blades, flakes, cores, and production by-products/waste is calculated. The summed contribution score defines Rank 1, Rank 2, and Rank 3; source nodes are added manually as Rank 4.
  3. Brainerd–Robinson diagnostics. Similarity/dissimilarity matrices, PERMANOVA-style tests, and PCoA visualisation are used to assess whether production-score ranks correspond to technological differentiation.
  4. Chronological split. Full-dataset ranks are attached to the 6100–5500 cal BC and 5500–4900 cal BC subsets.
  5. Rule-distance network modelling. Three rule sets are tested across distance thresholds from 100 to 650 km.
  6. Model comparison. The script exports metrics, centrality tables, rule-comparison figures, and non-parametric statistical tests.
  7. Source-confirmed networks. Sardegna / Monte Arci, Lipari, and Palmarola networks are extracted using confirmed source attribution and Rule A with Rule B fallback.
  8. Publication figures and tables. Maps, graphs, metric plots, ranked tables, centrality tables, source-filter logs, and lookup tables are exported to OUTPUTS/.

Rule sets

Rule setAllowed directed linksInterpretation
Rule A4-3, 3-2, 3-1, 2-1Parsimonious hierarchical model excluding direct source-to-intermediate links.
Rule B4-3, 4-2, 3-2, 3-1, 2-1Less restrictive model allowing direct links from sources to intermediate sites.
Rule C4-3, 3-2, 2-1Strict stepped hierarchical model.
