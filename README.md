CHRONOS is a reproducible R workflow for modelling Neolithic obsidian production, ranking, and circulation networks in the Central Mediterranean.
The repository contains the analytical R script, input datasets, map files, generated outputs, and an illustrated HTML report prepared for publication.

Open the CHRONOS HTML report
The report includes:

all figures generated by the script;
comments and interpretation notes for the figures;
a catalogue of output tables;
reproducibility and publication notes.

Project aims
CHRONOS was developed to model the production and circulation of obsidian across two Neolithic chronological intervals:
6100–5500 cal BC
5500–4900 cal BC
The workflow addresses four main questions:
How can archaeological sites be ranked according to their involvement in obsidian production and transformation?
How do different connection rules and distance thresholds affect reconstructed exchange networks?
Which network models provide the most parsimonious explanation for obsidian circulation?
How do source-confirmed networks differ for Sardinian / Monte Arci, Lipari, and Palmarola obsidian?

Analytical workflow
The workflow is implemented in `CHRONOS.R`.

1. Data loading and cleaning
The script reads the complete obsidian dataset and the two chronological subsets from the `RAWDATA/` folder.

Expected input files are:
```text
RAWDATA/obs_data_corr.xlsx
RAWDATA/obs_data_corr_6100_5500.xlsx
RAWDATA/obs_data_corr_5500_4900.xlsx
```

The script standardises:
site names;
longitude and latitude;
technological variables;
total artefact counts;
obsidian source-attribution columns.

The main technological variables are:
```text
blade
flake
core
by.products.waste
```

The source-attribution columns are:
```text
Sardegna
Lipari
Palmarola
```

2. Production-score ranking
The script calculates the contribution of each site to the total number of blades, flakes, cores, and production by-products/waste in the full dataset.
The production score is calculated as:
```text
Production_score =
blade.PN + flake.PN + core.PN + by.products.waste.PN
```

Sites are then assigned to three analytical ranks:
Rank	Interpretation	Criterion
Rank 1	Low production involvement / consumer sites	Below the 80th percentile
Rank 2	Intermediate production involvement	Between the 80th and 90th percentiles
Rank 3	High production involvement / producer sites	At or above the 90th percentile
Rank 4	Raw-material source nodes	Manually added source locations
The raw-material source nodes are:
Monte Arci
Lipari
Palmarola

3. Brainerd–Robinson diagnostic analysis
A Brainerd–Robinson-style similarity and dissimilarity matrix is calculated on the production-contribution values.
This stage is used diagnostically to evaluate whether the production-score ranks correspond to differences in multivariate technological space.
The script produces:
a Brainerd–Robinson similarity matrix;
a dissimilarity matrix;
a PERMANOVA-style test by rank;
a PCoA plot of the dissimilarity space.

4. Chronological splitting
Production ranks are calculated on the full dataset and then attached to the two chronological datasets.
This ensures that site ranking is consistent across both chronological intervals.

5. Network modelling
For each chronological interval, the script builds networks using three rule sets and multiple distance thresholds.
The rule sets are:
Rule set	Allowed connections	Interpretation
Rule A	`4-3`, `3-2`, `3-1`, `2-1`	Parsimonious hierarchical model
Rule B	`4-3`, `4-2`, `3-2`, `3-1`, `2-1`	Less restrictive model allowing direct source-to-intermediate links
Rule C	`4-3`, `3-2`, `2-1`	Strict stepped hierarchical model
Distance thresholds range from 100 km to 650 km, in 50 km steps.
For each model, the script exports:
edge lists;
geographic network maps;
global network metrics;
site-level centrality measures;
rule-distance sensitivity summaries.

6. Model comparison
The script compares rule sets across distance thresholds using:
number of edges;
number of components;
number of isolates;
largest component size;
first fully connected network for each rule set;
Friedman tests;
pairwise Wilcoxon signed-rank tests.

7. Source-confirmed networks
The workflow extracts source-confirmed networks for:
Sardegna / Monte Arci
Lipari
Palmarola
Rule A is used as the preferred parsimonious backbone. If Rule A produces an uninformative source-confirmed graph, the script falls back to Rule B.
The rule-selection process and fallback decisions are recorded in the output tables.

8. Publication map
The script also generates a numbered black-and-white map of all sites, together with a lookup table linking map IDs to site names and coordinates.

Running the workflow
Clone or download the repository, then open the project folder in R or RStudio.
The script expects the following structure:
```text
CHRONOS/
├── CHRONOS.R
├── RAWDATA/
├── MAP/
└── OUTPUTS/
```
Run the analysis from the repository root:
```r
source("CHRONOS.R")
```
or from the command line:
```bash
Rscript CHRONOS.R
```

All output folders are created automatically if they do not already exist.

Viewing the report locally
The main report is:
```text
CHRONOS_report.html
```

It can be opened locally in a web browser.
The report expects the `OUTPUTS/` folder to remain beside `CHRONOS_report.html`, because the figures are loaded using relative paths.
